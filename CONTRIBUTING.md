# Contributing to G-Nera

Merci de votre intérêt pour contribuer à G-Nera ! 🎉

## Code of Conduct

Ce projet respecte le principe de bienveillance et d'inclusion.

## Comment Contribuer

### 1. Signaler des Bugs

Avant de créer un rapport, vérifiez les [issues](https://github.com/louisondab05-wq/g-nera/issues) existantes.

**Format :**

```markdown
## Description
Description claire du bug

## Étapes pour reproduire
1. ...
2. ...

## Comportement attendu
...

## Comportement actuel
...
```

### 2. Proposer des Améliorations

Les suggestions sont bienvenues !

### 3. Pull Requests

1. **Fork le projet**

```bash
git clone https://github.com/YOUR_USERNAME/g-nera.git
cd g-nera
```

2. **Créer une branche**

```bash
git checkout -b feature/ma-fonctionnalite
```

3. **Commit vos changements**

```bash
git commit -m "feat: description"
```

4. **Push et créer une PR**

```bash
git push origin feature/ma-fonctionnalite
```

## Standards de Code

### TypeScript
- Utilisez des interfaces pour les types
- Activez strict mode
- Évitez `any`

### Python
- Suivez PEP 8
- Utilisez type hints
- Docstrings pour les fonctions

### React
- Composants fonctionnels
- TypeScript obligatoire
- Pas de prop-drilling excessif

## Questions ?

- Ouvrez une [Discussion](https://github.com/louisondab05-wq/g-nera/discussions)
- Consultez la [Documentation](./docs/)

Merci pour votre contribution ! 🚀
